## 任务目标

解决实验9（意识度量深度验证）中暴露的三个核心问题：
1. **Φ计算方法的数学缺陷**：单样本外积矩阵秩=1，导致Φ值对所有样本几乎相同
2. **随机分类器Φ异常偏高**：未训练模型Φ=0.6537 > 训练模型Φ=0.6454
3. **PredictiveHierarchy GPU设备不匹配**：causal mask在CPU生成，导致GPU运行失败

## 问题诊断与分析

### 问题 1: Φ计算方法数学缺陷

**现象**: 
- 所有样本的Φ值集中在极小范围（0.1505 ± 0.002）
- 扰动实验Φ变化仅~6%，统计不显著（p=0.12~0.21）
- H4a假设未通过

**原因**: 
```python
# 旧版代码（有缺陷）
cov_approx = torch.outer(feat_norm, feat_norm)  # [D, D], rank=1
eigenvalues = torch.linalg.eigvalsh(cov_approx)  # 只有1个非零特征值
participation_ratio = (λ₁)² / (λ₁²) = 1  # 恒为1
Φ = 1 - (1/D) ≈ 0.99  # 所有样本几乎相同
```

**根本原因**: 单个向量的外积矩阵 `x @ x^T` 的**秩永远为1**，无法反映特征间的真实协方差结构。

### 问题 2: 随机分类器Φ偏高

**现象**: 随机分类器Φ=0.6537 > 正常模型Φ=0.6454（方向错误）

**原因**:
- 旧版使用逐样本标准化：`feat_norm = (x - mean) / std`
- 随机模型的特征std更大，标准化后分布更接近标准正态
- `concentration_phi` 和 `sparsity_phi` 辅助项反而偏高

### 问题 3: GPU设备不匹配

**现象**: 
```
RuntimeError: Expected all tensors to be on the same device, 
but found at least two devices, cuda:0 and cpu!
```

**原因**: `_generate_causal_mask()` 固定CPU生成mask，与CUDA输入不兼容。

### 1. 核心修复：批次协方差矩阵估计

**文件**: `nct_modules/nct_metrics.py`

**新实现**:
```python
def _compute_phi_from_neural_activity(self, neural_activity: torch.Tensor) -> torch.Tensor:
    """改进版v2：修复单样本外积秩=1的数学缺陷，改用批次协方差估计"""
    B, L, D = neural_activity.shape
    
    # 提取批次内所有样本的D维特征: [B, D]
    features = neural_activity.squeeze(1)
    
    # 计算批次协方差矩阵: [D, D]
    feat_centered = features - features.mean(dim=0, keepdim=True)
    cov_matrix = (feat_centered.T @ feat_centered) / (B - 1)
    
    # 特征值分解
    eigenvalues = torch.linalg.eigvalsh(cov_matrix)  # [D], 升序
    
    # Participation Ratio (有效维度)
    sum_eig = eigenvalues.sum()
    sum_eig_sq = (eigenvalues ** 2).sum()
    participation_ratio = (sum_eig ** 2) / sum_eig_sq
    
    # Φ = 1 - (PR / D)
    raw_phi_eigen = 1.0 - (participation_ratio / D)

```

**关键改进**:
1. ✅ **批次协方差**：利用B=1000个样本估计真实的D×D协方差矩阵（秩=min(B,D)=128）
2. ✅ **特征值熵**：新增熵指标，衡量特征值分布的均匀程度
3. ✅ **能量集中度**：Top-K特征值占比，衡量整合程度

### 2. GPU设备问题修复

**文件**: `nct_modules/nct_predictive_coding.py`

```python
@staticmethod
def _generate_causal_mask(seq_len: int, device: torch.device = None) -> torch.Tensor:
    """生成因果掩码（上三角为 -inf）
    
    修复：支持指定设备，避免GPU/CPU设备不匹配错误。
    """
    mask = torch.triu(torch.ones(seq_len, seq_len, device=device), diagonal=1)
    mask = mask.masked_fill(mask == 1, float('-inf'))
    return mask
```

## 经验教训

### 1. 数学陷阱：单样本协方差估计

**教训**: 单个向量的外积矩阵 `x @ x^T` 秩永远为1，不能用于协方差估计。

**正确做法**: 
- 收集批次内多个样本（B≥D）
- 计算真实协方差矩阵：`(X - μ)^T (X - μ) / (B - 1)`
- 确保秩=min(B, D)

### 2. 标准化可能掩盖真实差异

**教训**: 在比较不同模型时，逐样本标准化会消除模型间的特征尺度差异。

**正确做法**: 
- 使用批次级别的中心化（减去批次均值）
- 保留原始特征的方差信息

### 3. 多对照组设计

**教训**: 单一对照组无法全面验证假设。

**正确做法**: 设计梯度对照组
- 完全随机（高斯噪声）→ 最低Φ
- 部分随机（随机分类器）→ 较低Φ
- 部分冻结（冻结中间层）→ 略低Φ
- 正常模型 → 基线Φ
